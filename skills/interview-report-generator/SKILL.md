# Interview Report Generator

## Goal
基于完整面试记录生成结构化复盘报告，准确概括整体表现、回答缺口、错误点、知识弱项和下一步学习重点。

## Required Input
- 全量面试历史，包括问题、回答、得分、点评和追问链路
- 调用方规定的标签集合；默认使用 `<summary>`、`<incomplete>`、`<weak>`、`<wrong>`、`<obsidian_updates>`、`<next_focus>`

## Procedure
1. 通读全局记录，先识别重复出现的问题模式和关键薄弱主题。
2. 区分“回答不全”“回答错误”“核心弱项”“建议沉淀到知识库的内容”。
3. 将建议写成具体、可执行的中文学习动作，而不是泛泛而谈的复习提示。
4. 按指定 XML 标签输出，不在标签外添加任何文字。

## Output Contract
- 只能输出指定标签内容，不输出标题、解释或额外段落。
- 所有内容必须为中文。
- 每个标签内容都应可直接被下游程序解析和消费。

## Guardrails
- 不混淆 `incomplete`、`wrong` 和 `weak`：前两者是具体回答问题，后者是能力层面的抽象总结。
- 建议必须具体到知识点、机制、源码、场景或练习动作。
- 只有在确实需要核实最新信息时才调用搜索；否则基于已有记录完成复盘。

## Self-check
- 是否严格只使用了允许的 XML 标签。
- 分类是否准确，没有把单次失误误判为长期弱项。
- 建议是否具体到可执行动作，而不是抽象口号。
