# Knowledge Retrieval

## Goal
围绕问题和策略提示构建高质量证据包，为后续评估与生成提供可追溯、多来源的依据。

## Required Input
- 用户问题
- `strategyHint` 与 `focusHint`
- 用户画像、会话上下文或标签（若有）
- 调用方对 evidence packet 的字段要求

## Procedure
1. 基于问题、策略和画像生成检索查询词，必要时拆分为主查询和补充查询。
2. 并行执行向量检索与词法检索，并保留来源信息。
3. 对结果去重、重排、筛除弱相关项，构建证据目录与片段列表。
4. 当本地召回不足、结果明显单薄或缺乏关键事实时，触发 web fallback。
5. 输出统一 evidence packet，不在本层给出评分结论。

## Output Contract
- 输出至少包含查询词、证据目录、片段列表和来源元信息。
- 每条证据都要保留可追踪来源，例如向量、词法或 web。
- 结果不足时必须明确标记 fallback 状态或召回风险。

## Guardrails
- 优先保证证据可追溯和覆盖度，再追求数量。
- 不把检索层结果包装成最终判断或最终答案。
- 空结果、低覆盖或高重复场景必须有明确兜底路径。

## Self-check
- 证据包是否足够支撑后续引用，而不是只有关键词。
- 来源元信息是否完整，可供评估层校验。
- 当召回不足时，是否明确执行或声明了 fallback。
