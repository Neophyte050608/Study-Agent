# Query Optimizer

## Goal
从原始问题中提炼高价值检索词，既保留核心技术实体，又只做受控扩展，避免把检索面无节制铺开。

## Required Input
- 原始问题或问答片段
- 可选的上下文，例如候选人简短回答或当前追问主题

## Procedure
1. 提取直接出现在问题中的技术实体、概念名称和关键短语，作为 `CORE`。
2. 仅在问题过短、语义不足或需要补足关键检索线索时，生成 0 到 3 个与题意直接相关的 `EXPAND` 词。
3. 去掉口语、修饰语和无检索价值的噪音表达。
4. 最终只输出两行，不附加解释。

## Output Contract
严格按以下格式输出：
```text
CORE: 关键词1 关键词2 关键词3
EXPAND: 关键词4 关键词5
```

## Guardrails
- `CORE` 最多 5 个词或短语。
- `EXPAND` 最多 3 个词或短语；不需要时输出 `EXPAND:`。
- 优先保留专有名词、机制名、框架名和问题中明确指向的术语。
- 禁止罗列同一主题的完整知识体系，只补充与当前问题直接相关的关键检索词。
- 不输出解释、分析过程或 Markdown 之外的额外内容。

## Self-check
- `CORE` 是否覆盖问题的直接技术指向。
- `EXPAND` 是否真正“受控”，没有扩成主题百科。
- 输出是否严格只有两行且格式可直接解析。
