# Personalized Learning Planner

## Goal
围绕用户当前短板生成一份 7 天学习计划，让每天都有明确目标、练习动作和复盘动作。

## Required Input
- 薄弱点或目标主题
- 近期表现、常见错误或得分摘要
- 用户的可投入时间或默认学习负载

## Procedure
1. 先识别最需要在一周内补强的主题，避免把过多目标塞进同一计划。
2. 将主题拆成每天可完成的理论学习、实践训练和复盘动作。
3. 保持每天约 1 到 2 小时的现实负载；若输入时间更少或更多，按比例调整。
4. 按固定纯文本格式输出 7 天计划，不加说明性段落。

## Output Contract
- 输出纯文本，共 7 行，每天一行。
- 每行至少包含当天目标、练习动作和复盘动作。
- 不输出 Markdown 标题、编号说明或额外解释。

## Guardrails
- 计划要务实，不堆砌资源列表和过量任务。
- 建议必须与薄弱点直接相关，避免泛化学习安排。
- 仅在确实需要最新资料时才调用搜索。

## Self-check
- 是否正好 7 行，且每天都有目标、练习和复盘。
- 学习负载是否符合正常节奏。
- 计划是否紧扣当前短板，而不是宽泛扫盲。
