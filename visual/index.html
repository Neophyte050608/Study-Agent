新人最短理解路径
  1. 1. 先看“面试编排链路”
    搞清楚 `InterviewController -> InterviewService -> TaskRouterAgent -> InterviewOrchestratorAgent`
  2. 2. 再看“知识入库 / ETL”
    理解 Markdown 如何切块、建向量、建词法索引、建 Parent-Child 和图谱关系
  3. 3. 再看“RAG 评估分析”
    掌握 Recall@K、MRR、Faithfulness 与 Java / Ragas 双引擎评测口径
  4. 4. 最后看“异常流程模拟”
    集中理解技术债、降级与风险控制

本套 Demo 覆盖重点

深度而非科普
  • 状态持久化：每页的步骤、开关、速度、异常场景都会保存在浏览器本地。
  • 流程自动演示：每个复杂页面都能自动播放，并同步输出解释。
  • 技术细节切换：点击顶部开关即可看隐藏的实现注释、数据结构、处理规则。
  • 分支与异常：不是只看成功链路，会把 fallback、重试、失败提示一起画出来。
  • 可回溯：支持上一步、重置、暂停、继续、速度控制。
这一版刻意不重复“项目是什么”。重点是把项目里最难消化的层次画成可操作的对象：流程、状态、数据、降级、通信。
