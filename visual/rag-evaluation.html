页面说明

评测口径区分
这个页面区分两种评测：`RetrievalEvaluationService` 评估“能不能把对的文档召回来”；`RAGQualityEvaluationService` 评估“拿到上下文后答案是不是忠实、相关、完整”。质量评测又分两套引擎：Java 本地评估，以及容器内 Ragas Python HTTP 评测服务。

关键实现点
基于实际代码

        
    
          
  • `RetrievalEvaluationService` 直接复用 `RAGService.buildKnowledgePacket(query, "", false)`，评测时显式关闭 Web fallback，确保指标衡量的是本地检索能力。
    • 
          
  • 检索评测的命中规则是：取前 5 条召回文本，只要文本包含任一 expected keyword，就算命中，并记录 firstHitRank 用来算 Recall@K 和 MRR。
    • 
          
  • 参数模板真实存在：`baseline-rrf`、`high-rerank`、`fallback-observe`，核心差异在 `rerankTopK` 和 `allowWebFallback`。
    • 
          
  • `RAGQualityEvaluationService` 会先走真实检索，再生成答案，然后并发计算 `faithfulness / answerRelevancy / contextPrecision / contextRecall`。
    • 
          
  • 质量评测支持 `java` 与 `ragas` 两种引擎；`ragas` 实际对应容器内 Python HTTP 服务，由 `RagasEvalClient` 通过 `serviceUrl/health` 探活、通过 `/evaluate` 发起批量评测；不可用时自动回退到 Java 本地评测流程。
    • 
          
  • 两类评测都支持留档：运行汇总表 + 逐样本明细表，并可通过 compare / trend 接口做 A/B 对比和趋势分析。
