# 智能刷题模块设计规格 (Smart Coding Practice Spec)

## 1. 模块定位与价值
“智能刷题”模块旨在为用户提供**个性化、自适应的对话式**技术训练环境。
它不同于传统的静态题库（如 LeetCode），而是能够**通过自然语言对话，结合用户的“历史学习画像”和“本地知识库”动态生成考题**。用户可以自由地在对话框中说出需求（如：“我想刷3道关于 Redis 缓存击穿的场景题”或“给我来一道动态规划的算法题”），并在用户提交答案后，提供多维度的 AI 评估，最后将练习结果闭环反馈到学习画像中。

## 2. 核心业务闭环 (The Core Loop)

### 2.1 对话式意图解析与推题 (Chat Intent & Recommend)
- **输入**：用户的自然语言输入（例如：“帮我出5道Java多线程的选择题” 或 “直接开始刷题”）。
- **处理**：
  - **意图识别**：通过大模型解析用户输入，提取出 `题型 (算法/选择/填空/场景)`、`题数`、`技术栈 (如 Redis/Java)` 和 `难度`。
  - **画像兜底**：如果用户只说“开始刷题”而未指定具体技术栈，系统将调用 `LearningProfileAgent` 查询用户当前的薄弱点（Weaknesses）作为推荐主题。
  - **动态出题**：`CodingPracticeAgent` 结合解析出的参数和用户的 `profileSnapshot`（画像快照），下发给 `RAGService`，在知识库的上下文中为你量身定制题目。
- **输出**：在聊天界面中以对话气泡的形式输出第一道定制化的题目描述与答题要求。

### 2.2 对话式答题与评估 (Submit & Evaluate)
- **输入**：用户在聊天框中输入的答案（可以是选择题选项、填空词汇、场景思路或具体的代码片段）。
- **处理**：
  - 用户的答案提交给后端的评估逻辑。
  - 大模型充当“智能判卷官”，根据不同题型执行差异化评估：
    - **算法题**：评估正确性、时间/空间复杂度、边界条件。
    - **场景题**：评估系统设计合理性、Trade-off 分析。
    - **客观题 (选择/填空)**：直接比对正确答案，并给出解析。
  - 生成评估结果：包含得分、详细评语（feedback）和知识点解析。
- **输出**：在对话流中展示 AI 的评语，并自动抛出下一道题（直到达到用户指定的题数）。

### 2.3 画像闭环更新 (Profile Feedback)
- 每次刷题会话结束后，`CodingPracticeAgent` 会自动构造一个或多个 `LearningEvent`（来源标记为 `LearningSource.CODING` 或 `LearningSource.PRACTICE`）。
- 该事件被推送到 `LearningProfileAgent`。
- 画像更新：错误点记录为“历史薄弱项”，连续答对的知识点记录为“熟练项”，实时反映在能力雷达图中。

## 3. 技术架构与 API 设计

### 3.1 后端依赖 (需改造)
目前的 `CodingPracticeAgent` 主要是为了算法题（`startPractice`/`submitPractice`）设计的。为了支持对话式多题型，需要进行以下改造：
1. **意图解析拦截器**：在 `execute` 方法前，增加对自然语言的意图提取（抽取题型、技术栈、题数）。
2. **多题型 RAG 提示词**：在 `RAGService` 中扩展 `generateCodingQuestion`，使其支持生成选择题、填空题和场景题的 Prompt。
3. **会话状态机扩展**：现有的 `CodingSession` 需要增加字段，记录“当前是第几题”、“总题数”、“当前题型”，以支持连续对话式的连贯刷题。

### 3.2 交互 API (对话式载体)
依然通过统一的任务派发接口进行交互，但更偏向对话流：
- **发送对话/指令**：
  ```json
  {
    "taskType": "CODING_PRACTICE",
    "payload": { 
      "action": "chat", 
      "sessionId": "xxx-xxx", 
      "message": "我要刷3道Spring Boot的场景题" // 或具体的答案 "选 A"
    }
  }
  ```
  后端根据 `sessionId` 的状态判断这句话是“开启新训练”还是“提交上一题的答案”。

### 3.3 前端页面设计 (`practice.html`)
摒弃传统的左右硬切分表单，采用类似 ChatGPT / Claude 的**沉浸式对话流 UI**：
- **主对话区**：
  - 用户的指令和答案作为右侧气泡。
  - AI 的出题、判卷结果、解析作为左侧气泡。支持 Markdown 渲染（针对代码块高亮）。
- **底部输入区**：
  - 核心输入框（支持多行文本、Tab 缩进，以便直接粘贴代码）。
  - 快捷指令气泡（例如提供快捷按钮：“换一题”、“结束本次刷题”、“查看解析”）。
- **右侧/悬浮抽屉 (状态面板)**：
  - 实时显示当前刷题进度（例如：2/5 题）。
  - 实时雷达图或薄弱点提示（调用画像数据）。

## 4. 后续落地计划 (Phase)
- **Phase 1 (后端重构)**：升级 `CodingPracticeAgent` 与 `RAGService`，引入大模型意图识别，支持从自然语言中提取刷题配置，并扩展多题型的 Prompt 生成与评估逻辑。
- **Phase 2 (前端对话流)**：创建 `practice.html` 页面，搭建沉浸式的聊天交互界面，完成与后端的对话流联调。
- **Phase 3 (MCP 沙盒执行 - 仅限算法题)**：结合当前的 MCP 最小闭环，如果遇到算法题代码，自动将代码通过 `code.execute` (MCP工具) 发送到沙盒运行，验证真实编译情况。
